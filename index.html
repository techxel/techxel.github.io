Mount ex-disk

Posted on 2020-08-11 Edited on 2020-08-12

Our SDCard just 16G, even though yours are 128G, it’s not big enough for us to build the NAS. Sometime we have extra hard disk from old computers which can be used now.

First we should know what the storage format they are, mostly the format is NTFS, exFAT.

  • NTFS: mainly be used in Windows, it can be read on mac, but can’t be wirte

  • exFAT: can be used the both platforms, so some one who used mac, the format of their portable hard disk must be exFAT

maybe some one also used FAT32 which is outdated now. so we didn’t talk it here

If we connect the disk to our PI, we can find the mount information in /media/pi, which is mounted by the PI operating system automatically

When we want to build a NAS, commonly we’ll mount it in /mnt, so first we unmount it from /media/pi, then mount to /mnt, the command like this:

sudo umount /media/pi
sudo mount /dev/sda1 /mnt

our ex-disk is commonly under the /dev/sda, if you have just one partition then the /dev/sda1 must be your disk

But the PI didn’t support exFAT defaultly, we must install some includings first

sudo apt-get install exfat-fuse

For the format of NTFS, PI just can read but can’t write in it, we also need install some includings and open the module

sudo apt-get install ntfs-3g
sudo modprobe fuse

At last, we need write the record to /etc/fstab, something like this

echo '/dev/sda1 /mnt ext4 defaults 0 0' >> /etc/fstab

be careful the >>, you can’t write >

Notice:

Once the record was written in /etc/fstab, you can’t remove the disk when you reboot the computer, or you can’t login in the system. If you want to avoid this situation, you can configure like this:

/dev/sda1 /mnt auto defaults,nofail,x-systemd.device-timeout=1,noatime,umask=0000 0 0

Instal Raspberry system by using Mac terminal

Posted on 2020-07-30

first of all: format the SDCard, if your SDCard is totally a new one, you can skip this step

you can use SD Card Formatter or Disk Utility which can be found in system Applications

step 2: download operating system

I highly recommend you to install the official operating system which is more friendly for beginner

you can download them from the following link, if you are in China, I suggest you use some BT tools, or use proxy tools such as ss, v2ray or trojan

https://www.raspberrypi.org/downloads/raspberry-pi-os/

step 3: unmount the disk

the /dev/disk2 is your SDCard, now you can unmount its partition /dev/disk2s1 and write into the image which you downloaded before

in Linux distros, we always use the command umount, but in Mac here, we use unmount

step 4: write the image to rdisk2

  • if= means the path of your operating system image
  • of= meams the path of your SDCard

now you can check the disk, one more partition named disk2s2
